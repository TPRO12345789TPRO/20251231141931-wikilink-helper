# Release v1.3.0 - GitHub Automation & Architecture Refactor

## Summary
Release v1.3.0 introduces a major architectural refactor to support site-specific handlers, with **GitHub.com** being the first implementation. It enables "Auto-Navigate" mode, which automates file searching on GitHub when clicking a wikilink. This release also includes extension icons and a privacy policy.

## Implementation Details

### 1. Architectural Refactor
-   **Structure**: Moved logic to `src/core` and `src/handlers`.
-   **Handler Pattern**: Implemented `BaseHandler` and `GitHubHandler` to support extensible site-specific logic.
-   **Entry Point**: `src/main.js` replaces `content.js`, dynamically selecting the appropriate handler.

### 2. GitHub Automation
-   **Logic**: `GitHubHandler` detects clicks on wikilinks.
-   **Action**: If "Auto-Navigate" is enabled in the popup:
    1.  Dispatches `t` keypress (opens file finder).
    2.  Inputs the wikilink ID.
    3.  Dispatches `Enter` (with 500ms delay) to confirm navigation.
-   **ZIDs**:
    -   `20251231155650`: Initial refactoring plan.
    -   `20251231160706`: Automation logic implementation.

### 3. Security & Polish
-   **Scope**: Initially restricted to GitHub, but reverted to `<all_urls>` to maintain global "Copy" functionality (ZID `20251231161433`).
-   **Icons**: Added 16px, 48px, and 128px icons to `assets/`.
-   **Privacy**: Added `PRIVACY.md` per Web Store requirements.

## Decision Log
-   **Reverting Scope**: We decided to allow the extension to run on all sites (`<all_urls>`) so that the core "Copy ID" feature works universally, while restricting the advanced "Auto-Navigate" feature solely to GitHub via the handler check.
